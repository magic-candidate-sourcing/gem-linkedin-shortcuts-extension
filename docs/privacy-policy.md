# Privacy Policy - Gem LinkedIn Shortcuts

Last updated: March 26, 2026

Gem LinkedIn Shortcuts is an internal recruiting workflow extension.

## What data the extension accesses

- Profile context from supported pages when a user triggers an action, including LinkedIn, Gem candidate/project, and GitHub profile URLs, visible names, and visible contact details.
- User-configured extension settings (shortcuts, backend URL, and optional defaults).
- User-entered action data such as project selection, notes, reminder dates, and custom-field values.
- Action logs generated by extension/backend for troubleshooting.

## What data the extension does not store

- Gem API keys and Ashby API keys are not stored in the extension.
- Chrome Web Store builds should not ship org-wide backend shared tokens inside the extension bundle.
- Credentials remain on your organization's backend service.

## How data is used

Data is used only to execute user-initiated recruiting actions (for example adding a candidate, uploading to Ashby, adding notes/reminders, and opening candidate records) and to provide debugging logs when your team investigates failures.

## Data sharing

- The extension sends requests to your organization's backend endpoint.
- The backend then calls Gem and Ashby APIs as required.
- Data is not sold.

## Data retention

- Local extension logs are stored in browser local storage until cleared or rotated by log limits.
- Backend-side retention is controlled by your organization's backend and hosting/logging policy.

## Security

- Backend routes are allowlisted to extension-supported actions.
- Optional backend shared-token validation and extension-origin allowlisting can be enabled.
- Sensitive fields (token/key/secret/password-like values) are redacted in logs where possible.

## User controls

Users can:
- Enable/disable the extension.
- Change or reset shortcuts and settings.
- Clear local extension logs.

## Contact

For questions about this extension or this policy, contact your internal extension owner/admin.
